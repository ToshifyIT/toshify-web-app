Model Context Protocol server para chatbots. Acceso completo (CRUD) a leads y hireflix_historico.


Autenticacion

Este servidor requiere una API key para conectarse. Las API keys se gestionan desde la app de Toshify en Administracion > Integraciones > MCP.

1
Obtener tu API key
Un administrador genera la API key desde la app y te la comparte.
2
Conectar via SSE
Usar el endpoint /sse con el header x-api-key o query param ?apiKey=
3
Configurar tu cliente MCP
{
  "mcpServers": {
    "toshify": {
      "url": "/sse?apiKey=TU_API_KEY"
    }
  }
}

Permisos

Cada API key tiene permisos granulares. Solo ve las herramientas para las que tiene permiso.

Buscar y ver leads

Herramientas - Leads

buscar_leads leads:read
Buscar leads por nombre, DNI, email, estado, sede u otros filtros. Devuelve una lista paginada.
ParametroTipoRequeridoDescripcion
searchstringNoBuscar en Nombre Completo, Email o DNI
estadostringNoFiltrar por Estado de Lead (ej: Nuevo, Contactado, En proceso)
agentestringNoFiltrar por Agente asignado
sedestringNoFiltrar por Sede (ej: Buenos Aires, Cordoba)
desdestringNoFecha desde (ISO 8601, ej: 2025-01-01)
hastastringNoFecha hasta (ISO 8601, ej: 2025-12-31)
pagenumberNoPagina (default: 1)
limitnumberNoCantidad por pagina (default: 20, max: 100)

Lo que le decis al chatbot:
"Busca todos los leads que se llamen Juan"

buscar_leads({ search: "Juan" })

      "estado_de_lead": "Contactado",
      "sede": "Buenos Aires",
      "agente_asignado": "Maria Garcia",
      "fecha_creacion": "2025-06-15T10:30:00Z",
      "fase_de_preguntas": "Entrevista"

actualizar_lead leads:update
Actualizar campos de un lead existente. Solo modifica los campos que envies, el resto queda igual.
ParametroTipoRequeridoDescripcion
idstring (UUID)SiID del lead a actualizar
camposobjectSiCampos a actualizar (mismos campos permitidos que crear_lead)

Ejemplo 1: Cambiar estado

Lo que le decis al chatbot:
"Cambia el estado de Juan Perez a Contactado"
Tool call:
actualizar_lead({
  id: "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  campos: {
    "estado_de_lead": "Contactado"
  }
})
Respuesta:
{
  "mensaje": "Lead actualizado exitosamente",
  "camposActualizados": ["estado_de_lead"],
  "lead": {
    "id": "a1b2c3d4-...",
    "nombre_completo": "Juan Perez",
    "estado_de_lead": "Contactado",
    ...
  }
}

Ejemplo 2: Actualizar varios campos a la vez

Tool call:
actualizar_lead({
  id: "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  campos: {
    "sede": "Cordoba",
    "agente_asignado": "Pedro Martinez",
    "phone": "+5493511234567",
    "fase_de_preguntas": "Documentacion"
  }
})

Ejemplo 3: Si mandas un campo que no existe

Tool call:
actualizar_lead({
  id: "a1b2c3d4-...",
  campos: {
    "estado_de_lead": "En proceso",
    "campo_inventado": "valor"
  }
})
Respuesta (actualiza lo valido, avisa lo rechazado):
{
  "mensaje": "Lead actualizado exitosamente",
  "camposActualizados": ["estado_de_lead"],
  "camposRechazados": ["campo_inventado"],
  "lead": { ... }
}

Seguridad

  • Este servidor solo tiene acceso a las tablas leads y hireflix_historico. Nada mas.
  • No tiene acceso a conductores, vehiculos, facturacion, ni ninguna otra tabla del sistema.
  • Cada API key tiene permisos granulares - podes crear una key de solo lectura, o una que solo pueda crear pero no borrar, etc.
  • Las API keys se pueden desactivar en cualquier momento desde la app de administracion.
  • Se registra la ultima vez que se uso cada API key.
  • Las acciones de eliminar son irreversibles. Usa con cuidado.
